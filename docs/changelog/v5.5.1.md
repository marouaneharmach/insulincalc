# Changelog v5.5.1 — 2026-03-31

## Contexte
Corrections critiques du suivi injection et améliorations UX repas/lipides.

## Corrections critiques

### IOB (Insulin On Board) — bug de sécurité
- **Problème** : Le calcul IOB ne lisait que `doseReelle` et `doseInjected`. Les doses modifiées via DayTimeline (sauvées dans `doseActual`) étaient invisibles au calcul.
- **Fix** : Lecture unifiée `doseActual ?? doseReelle ?? doseInjected` avec `??` (nullish coalescing) — priorité à la valeur la plus récente.
- **Fichiers** : `ConsultationScreen.jsx` (3 locations), `DayTimeline.jsx` (saveDose)

### Dose 0u
- `saveDose` dans DayTimeline écrit maintenant dans `doseActual` ET `doseReelle` pour cohérence.

## Nouvelles fonctionnalités

### Suivi injection amélioré (DayTimeline)
- **Comparaison dose** : Affichage côte à côte "Suggérée → Réelle" avec delta coloré quand les valeurs diffèrent
- **Édition inline** : Bouton ✏️ explicite à côté de chaque dose (meilleure tapabilité mobile)
- **Injection manuelle** : FAB "+" en bas à droite pour ajouter une injection oubliée (Correction / Autre) avec horodatage

### Mode Aliments par défaut (MealInput)
- **Avant** : Mode "Expert" (saisie manuelle glucides) par défaut
- **Après** : Mode "Aliments" (base de données) par défaut, onglets réordonnés

### Calcul lipides (MealInput + ConsultationScreen)
- Calcul automatique des lipides en grammes (`~Xg`) depuis les sélections alimentaires
- Mapping automatique vers le niveau qualitatif (aucun/faible/moyen/élevé) pour le moteur clinique
- Seuils : 0-5g → aucun, 5-15g → faible, 15-30g → moyen, >30g → élevé
- Champ `totalLipides` ajouté aux entrées journal
