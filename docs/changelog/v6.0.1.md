# Changelog v6.0.1 — 2026-04-11

## Contexte
Sécurisation de la chaîne de release v6.x avec traçabilité de version, diagnostic applicatif et mécanisme de mise à jour forcée côté navigateur.

## Changements

### Version & build
- `APP_VERSION`, `BUILD_ID` et `BUILD_DATE` deviennent des métadonnées injectées au build.
- La version et le build sont visibles dans le header et dans le diagnostic.

### Mise à jour navigateur
- Génération d’un `version.json` et d’un `sw.js` alignés sur le build Vite.
- Détection d’une version distante plus récente au chargement.
- Bandeau d’update + action “Actualiser maintenant”.
- Bouton “Forcer la mise à jour” dans Réglages → Diagnostic.

### Non-régression
- Suite dédiée `test:regression` pour la version, le moteur clinique, la migration et le panneau de diagnostic.
- Workflow GitHub Pages durci avec `lint` puis `test:regression` avant les tests complets et le build.
