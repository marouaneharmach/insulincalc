# Changelog v5.6.0 — 2026-03-31

## Contexte
Refonte du suivi injection post-repas — planning, notifications, tracking dans la timeline.

## Changements

### Planning injection (ConsultationScreen)
- **Plan d'injection post-confirmation** : Après "Confirmer", affichage d'une carte récapitulative avec :
  - ✅ 1re injection : Xu — maintenant (HH:MM)
  - ⏰ 2e injection : Yu — dans 45/60min (à HH:MM) *(si bolus fractionné)*
  - 📊 Contrôle glycémie — à HH:MM (2h après)
  - Bouton "Nouvelle consultation" pour réinitialiser
- **Persistance des détails split** : `splitImmediate`, `splitDelayed`, `splitDelayMinutes` sauvegardés dans chaque entrée journal

### Notifications (App.jsx)
- **Rappel 2e injection** : Notification automatique au bout de `splitDelayMinutes` pour les bolus fractionnés
- **Rappel glycémie post-repas** : Notification à 2h après chaque enregistrement
- **Fix** : `doseReelle` n'est plus écrasée par `doseSuggeree` dans `onSaveToJournalV5`

### Suivi dans DayTimeline
- **Steps split** : Pour les entrées fractionnées, affichage des deux étapes avec statut toggle (tap pour marquer fait)
- **Glycémie post** : Saisie inline déjà disponible (confirmée fonctionnelle)
